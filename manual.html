Git Workflow

  • 1) Master - stable production
  • 2) Develop - main branch for development
    • 2.1) feature#1 - code should be checked, tested
  • 3) Merge develop with master // git checkout master // git merge develop
  • 4) Create Release branch from develop and test all features
  • 5) Merge master with release
  • 6) HotFixes: (fix urgent bug on production)
    • Create new branch from master
    • Fix, test, merge back to master
    • merge with develop
